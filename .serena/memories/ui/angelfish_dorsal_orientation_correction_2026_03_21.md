2026-03-21: Added Angelfish per-asset GLB orientation correction in `src/components/DetailedFish.ts`. School and hero both use `schoolCorrectionQuaternion` / `heroCorrectionQuaternion` = `[-0.70710678, 0, 0, 0.70710678]` (rotate X by -90deg) while keeping forward axis `[1,0,0]`. Rationale: inspected `fish-angelfish-school.glb` and `fish-angelfish-hero.glb`; geometry bounds indicate forward already aligns to +X while dorsal was effectively on +Z, so whole-model correction rotates dorsal to +Y and ventral to -Y without turning nose tail-first. School asset is a single mesh; hero asset is two anonymous mesh nodes, so no robust named-child dorsal fin correction path exists in runtime. Added tests in `src/components/DetailedFish.test.ts` to assert Angelfish correction data exists and that school/hero corrections map forward to +X, dorsal to +Y, and ventral to -Y.
