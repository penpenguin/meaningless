2026-03-20: Reworked `src/components/Aquascaping.ts` planted layout so `plantClusterDefinitions` behave as zone definitions instead of near-final anchors. Each zone now defines `coreCount`, `satelliteCount`, `offshootCount`, and radius bands; `sampleZonePlacements()` uses seeded Poisson-like rejection sampling to create irregular `core + satellites + offshoots` placements with per-instance lane, azimuth, tilt, and hue variation. Default planted zones were reduced to two foreground zones plus rear/midground masses, yielding 28 planted placements instead of a denser regular grid. Species mixing is now deterministic enough to keep broad colonies readable: cores use the dominant species, while some satellites/offshoots deliberately flip to the secondary species so `left-rear` includes sword+fan and `driftwood-backfill` includes fan+sword. `substratePlantAnchors` are now derived from sampled placements with IDs matching placement IDs and use local colony density, cluster kind, depth lane, and tilt to vary mound radius/height and scoop direction/depth. Background plant tint floors were lifted (`getPlantTint` / `tonePlantColor`) so repeated background foliage reads as green mass instead of black lines. Added tests in `src/components/Aquascaping.test.ts` for zone-driven colony sampling, species/lane/lean diversity, foreground spacing irregularity, substrate anchor variability, and the brighter background tint floor. Verified with `npm run test`, `npm run typecheck`, and `npm run build`.
